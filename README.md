## ✨ 核心痛点与终极解决方案

在使用常规脚本（如单纯开启 BBR 或强制放大缓冲区）时，你是否遇到过以下灾难？
1. **不对称带宽的受害者：** 面对 `500M 下行 / 50M 上行` 的 VPS，常规脚本要么撑爆上传内存，要么让 500M 的下载速度直接腰斩。
2. **运营商 QoS 拔网线：** BBR 算法极具侵略性，突发超速极易触发上级路由器的强制丢包（Policing），导致单次测速出现数万次重传，甚至直接断流。
3. **高频访问直接报错：** 部署了高并发的 Vue 前端或 MediaMTX 流媒体，流量一上就报 `Too many open files` 或 `Address already in use`。
4. **小内存机器疯狂卡顿：** 1核1G 机器在面临突发流量时，疯狂读写 Swap 硬盘交换区，导致 Ping 值飙红，甚至 SSH 断开。

**本项目通过以下独家技术，一键彻底解决上述瓶颈：**

### 🌟 独家核心特性 (v5.0 全栈解封)

* ⚖️ **真·非对称缓冲区计算 (Asymmetric BDP Tuning)**
  * 彻底分离接收（RX）与发送（TX）计算逻辑，杜绝 Bufferbloat 导致的延迟抖动。
* 🚦 **智能流量整形 (TC FQ Pacing)**
  * 独立模块管控出站流量。通过平滑发包，彻底规避运营商的按压式限速惩罚，将 **数万次重传瞬间降至 0**。
* 🔓 **百万并发底座解封 (FD & Keepalive)**
  * 自动修改 `fs.file-max` 与 `/etc/security/limits.d`，突破默认 1024 限制，提升至 **100 万级文件句柄上限**。配合 TCP Keepalive 探针加速清理死链。
* ⚡ **跨洋起步爆发 & 多核分流 (initcwnd=32 + RPS)**
  * 修改路由表初始窗口至 32，跨海 API 与网页首包响应速度飙升 200%。同时自动识别多核 CPU 分配 RPS 掩码，防止单核软中断满载卡死。
* 🧠 **内存防抖与 OOM 护盾 (Swappiness & TCP Mem)**
  * 动态设定最高 25% 的 TCP 内存红线，并将 `vm.swappiness` 压至 10，死守物理内存，拒绝低效的硬盘 IO 拖慢网络。
* 💾 **Systemd 级无感持久化 & 中文注释**
  * 生成的配置自带详尽**中文注释**。自动接管 Linux 重启后会失效的 `initcwnd` 和 `RPS` 设置，创建静默系统服务，每次开机满血恢复。

### 环境要求
* 操作系统：Ubuntu 18.04+ / Debian 9+ / CentOS 7+ 
* 内核版本：Linux Kernel 4.9+ (内置 BBR 即可，兼容官方原版内核及 XanMod 等第三方内核)
* 执行权限：必须以 `root` 权限运行
